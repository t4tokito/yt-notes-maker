# AI Study Planner - Feature Plan

## Overview
AI Study Planner — User apna syllabus/topics + exam date daale, AI automatically daily study schedule bana de with progress tracking. Unique feature: kisi free study app mein nahi hai properly.

## What the Feature Does
1. User enters: topics to study, exam date, hours per day available
2. AI generates: day-by-day study plan with topic allocation
3. User can: mark days complete, track progress, see calendar view
4. Smart: weaker/longer topics get more days, revision days auto-added

## Files to Create/Modify

### Backend
- **`server/index.js`** — Add `POST /api/study-plan` endpoint
  - Request: `{ topics: string[], examDate: string, hoursPerDay: number }`
  - Response: `{ plan: StudyDay[], summary: string }`
  - StudyDay: `{ day: number, date: string, topics: string[], hours: number, type: "study" | "revision" | "rest" }`

### Frontend — New Screen
- **`app/study-planner.tsx`** — New screen (reuses `ai-tools.tsx` pattern)
  - Phase flow: setup → loading → plan → active
  - Setup: topic input (add/remove chips), exam date picker, hours selector
  - Plan view: scrollable day cards with topic pills
  - Active mode: mark today complete, progress bar

### Frontend — Modified Files
- **`app/ai-tools.tsx`** — Add Study Planner card (5th tool, replaces placeholder)
- **`app/create.tsx`** — Already has "AI Study Tools" card, link to study-planner
- **`app/_layout.tsx`** — Add `<Stack.Screen name="study-planner" />` route
- **`lib/api.ts`** — Add `generateStudyPlan()` function
- **`lib/studyPlan.ts`** — New file: Firestore CRUD for study plans

### Firestore Collection
- **`users/{uid}/studyPlans/{planId}`** — Stored plans
  ```
  {
    id: string,
    topics: string[],
    examDate: string,
    hoursPerDay: number,
    plan: StudyDay[],
    summary: string,
    completedDays: number[],
    created_at: number
  }
  ```

## Implementation Steps

### Step 1: Backend Endpoint
Add to `server/index.js`:
- `POST /api/study-plan` endpoint
- OpenRouter prompt that takes topics, exam date, hours/day → returns JSON study plan
- Parse JSON response, validate structure

### Step 2: API Client
Add to `lib/api.ts`:
- `generateStudyPlan(topics, examDate, hoursPerDay)` function
- Follow same pattern as `generateQuiz()`

### Step 3: Firestore CRUD
Create `lib/studyPlan.ts`:
- `saveStudyPlan(plan)` — Save to Firestore
- `getStudyPlans()` — Get all plans
- `getStudyPlan(id)` — Get single plan
- `updateStudyPlan(id, fields)` — Update completedDays
- `deleteStudyPlan(id)` — Delete plan

### Step 4: Study Planner Screen
Create `app/study-planner.tsx`:
- **Setup Phase**: Topic chips input, date picker (react-native community date picker or simple TextInput), hours selector (1-8 slider/buttons)
- **Loading Phase**: Progress bar with "Creating your study plan..." text (reuse quiz loading pattern)
- **Plan Phase**: Day cards in ScrollView, each with topic pills and hours
- **Active Phase**: Today highlighted, "Mark Complete" button, overall progress bar

### Step 5: Navigation & Integration
- Add route to `_layout.tsx`
- Update `ai-tools.tsx` to link to study-planner
- Update `create.tsx` if needed

## OpenRouter Prompt Design

```
System: You are an expert study planner. Create a detailed day-by-day study schedule.

User: Create a study plan with these details:
- Topics: Biology (Cell Division, Genetics), Physics (Newton's Laws, Motion)
- Exam Date: July 10, 2026
- Hours per day: 4
- Today's date: June 24, 2026

Rules:
1. Distribute topics evenly across available days
2. Add revision days every 3-4 days
3. Last 2 days should be pure revision
4. Longer topics get proportionally more time
5. Include rest days if schedule permits

Respond with ONLY valid JSON:
{
  "plan": [
    { "day": 1, "date": "2026-06-24", "topics": ["Biology: Cell Division"], "hours": 4, "type": "study" },
    ...
  ],
  "summary": "14-day plan covering 4 topics with 2 revision cycles"
}
```
