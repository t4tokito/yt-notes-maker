# Version Check System

## Goal
Backend and frontend share a version constant. On app open, frontend checks server version — if mismatch, show "Update Available" popup.

## Files to change

### 1. `server/index.js` — Add endpoint
- Add `const VERSION = "1.0.0"` at top
- Add `app.get("/api/version")` returning `{ version: VERSION }` near `/api/health`

### 2. `config.ts` — Add frontend version
- Add `export const APP_VERSION = "1.0.0"`

### 3. `lib/api.ts` — Add check function
- Add `checkVersion()` that fetches `/api/version` and returns the version string

### 4. `components/VersionCheckModal.tsx` — New file
- Modal with "Update Available" title, description, and Update button
- Update button opens download page via `Linking.openURL()` (app distributed via website, not Play Store)
- URL stored as constant in the component or config.ts
- Uses existing theme tokens (`colors.card`, `colors.accent`, etc.)

### 5. `app/_layout.tsx` — Wire it up
- In `AppShell`: `useEffect` calls `checkVersion()`, compares with `APP_VERSION`
- Render `<VersionCheckModal />` alongside `<UsernamePrompt />`

## Bump workflow
When releasing, update both `VERSION` in `server/index.js` and `APP_VERSION` in `config.ts`.

## Verification
1. Start backend — `curl localhost:3000/api/version` returns `{ version: "1.0.0" }`
2. App opens with matching version → no popup
3. Change one VERSION to "2.0.0" → popup appears on next app open
4. Tap Update → opens download page
